# Codex Context Min

## 项目背景

- 本仓库是 `Tysl.Ai` 的重构版新仓库，不是旧项目直接迁移。
- 产品定位固定为地图中心型桌面监控系统。
- 主界面长期保持“地图主视图 + 右侧详情抽屉 + 底部联动区”。

## 当前开发状态

- 真实 ACIS 内核已接入。
- 平台设备权威源主路径为 `AcisKernelPlatformSiteProvider`。
- 地图宿主已切到 `WebView2 + 高德 JSAPI 2.0`。
- 本地地图前端资源位于 `src/Tysl.Ai.UI/Web/amap/`。
- 已完成静默巡检、运行态、截图留痕、企业微信派单与恢复首版。
- V2 第一轮已聚焦坐标治理与未落图点位管理。

## V2 第一轮坐标治理规则

- 顶部同时显示总点位、已落图、未落图。
- “总点位”是全部点位，不等于地图当前已显示的点位数。
- 当前地图只显示具备可用显示坐标的点位。
- 已落图点位可来自平台原始坐标，也可来自本地手工坐标。
- 未落图点位需要在本地补充信息中继续治理。
- 地图区明确展示“当前地图显示 X / 总点位 Y”。
- 未落图治理入口保持轻量，不回退成传统后台大表格页。

## 当前降级策略

- 无 `configs/acis-kernel.json`：应用可运行，但无平台点位，静默巡检进入跳过模式。
- 无 `configs/amap-js.json`：应用可运行，中央区域显示“地图未配置”。
- 无 `configs/dispatch.json`：应用可运行，派单链路保持未配置状态。
- webhook 单条发送失败：只影响当前记录，不影响其他点位巡检。

## 当前展示约束

- 点位默认只显示“小图标 + 一行名称”，名称规则为别名优先、设备名兜底。
- hover 不再弹大面板。
- 点击 marker 只高亮当前点位，并联动右侧详情抽屉。
- 右侧详情抽屉继续固定为“基础信息 / 运行态 / 派单与恢复 / 坐标与平台 / 维护信息”。
- UI 不显示调试、堆栈或诊断信息。

## 当前边界

- 当前只支持企业微信 webhook。
- 不接复杂工单系统、审批流和复杂报表。
- 不恢复卡片墙式点位展示。
- 未落图治理保持轻量入口，不做满屏 DataGrid 后台页面。
