# UI 布局职责

## 顶部状态条

- 展示系统名称、点位统计和最后刷新时间
- 展示轻量“平台状态”徽标，用于显示“平台已连接 / 平台未连接 / 平台数据暂不可用”
- 不展示 token、签名、异常堆栈或诊断日志

## 左侧筛选栏

- 提供搜索和 `全部 / 异常 / 已监测 / 已派单` 快速筛选
- 仅影响地图点位与底部异常缩略条的显示范围
- 不承载平台配置表单

## 中央地图主视图

- 持续承载平台权威点位与本地补充信息合并后的地图标记
- 当前轮次允许直接消费后端输出的显示坐标，但不在后端做高德转换
- 后续接入真实地图宿主时，由前端高德 JSAPI 负责处理平台原始坐标转换
- 地图点击拾取仍然只服务于“手工坐标补录”

## 右侧详情抽屉

- 继续展示：
  - 设备编码
  - 设备名
  - 别名
  - 是否监测
  - 维护信息
  - 坐标来源
  - 坐标状态
  - 平台状态摘要
- 坐标来源至少区分：
  - 平台原始
  - 本地手工
  - 暂无坐标
- 不提供平台点位新增能力
- 不允许直接编辑平台主档字段

## 编辑弹窗

- 仅编辑本地补充信息
- `device_code` 和 `device_name` 只读
- 允许首次保存当前 `device_code` 的本地补充记录
- 手工坐标仍作为平台无坐标时的兜底来源
- 不允许新建平台点位

## 底部异常缩略条

- 继续展示由平台快照状态驱动的异常摘要
- 点击后切换当前点位与右侧详情
- 保持轻量，不扩展成任务面板
