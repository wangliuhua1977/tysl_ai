# UI 布局职责

## 顶部状态条

- 展示系统名称、统计卡片和最后刷新时间
- 展示轻量平台状态徽标
- 不显示 token、签名、堆栈或诊断日志

## 左侧筛选栏

- 提供搜索和 `全部 / 异常 / 已监测 / 已派单` 快速筛选
- 仅影响地图点位与底部异常缩略条
- 不承载平台配置表单

## 中央地图主视图

- 继续使用 `WebView2 + 高德 JSAPI 2.0`
- 通过本地宿主页 `UI/Web/amap/` 加载地图
- 点位默认只显示：
  - 简洁 marker
  - 极小状态角标
  - 一行名称标签
- 名称标签规则：
  - 有别名显示别名
  - 无别名显示设备名
- 默认不展开所有摘要卡片
- 只有 hover / click marker 时才显示轻量摘要卡片
- 摘要卡片只显示：
  - 点位名称
  - 当前状态
  - 监测状态
  - 简短运行/处置摘要
- 状态角标可反映：
  - 待派单
  - 已派单
  - 冷却中
  - 待人工恢复
  - 已恢复

## 右侧详情抽屉

- 继续承载：
  - 设备编码
  - 设备名
  - 别名
  - 监测状态
  - 平台原始坐标
  - 本地手工坐标
  - 当前显示坐标
  - 坐标来源
  - 维护信息
  - 备注
- 运行态信息包括：
  - 最近巡检时间
  - 巡检结果
  - 预览解析结果
  - 平台状态摘要
  - 连续失败次数
  - 最近截图时间
  - 最近截图预览
  - 运行态摘要
- 派单处置信息包括：
  - 当前派单状态
  - 最近触发时间
  - 最近派单时间
  - 冷却到期时间
  - 恢复状态
  - 恢复时间
  - 恢复摘要
  - 手工确认恢复入口
- 不在首页堆大量处置按钮

## 编辑弹窗

- 只编辑本地补充信息
- `deviceCode` 和 `deviceName` 只读
- 支持地图点击回填手工经纬度
- 保存后触发地图重新渲染

## 底部异常缩略条

- 展示异常点位的最近运行态摘要
- 优先展示最近截图留痕
- 可展示：
  - 已派单
  - 冷却中
  - 待人工恢复
  - 已恢复
- 点击后切换当前点位并同步地图高亮
- 保持轻量，不扩展成任务面板
