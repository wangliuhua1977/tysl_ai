# UI 布局职责

## 顶部状态区

- 展示系统名称、状态胶囊和克制的统计卡片。
- 统计卡片首版固定包含：
  - 总点位
  - 已落图
  - 未落图
  - 已纳管
  - 最近刷新
- 顶部不展示日志、调试信息或诊断状态。

说明：

- “总点位”是全部点位，不代表地图上都能看到。
- “已落图”是具备可用显示坐标、可以参与地图展示的点位。
- “未落图”是没有可用显示坐标、需要继续治理的点位。

## 左侧筛选区

- 提供搜索和 `全部 / 异常 / 已纳管 / 已处置 / 未落图` 快速筛选。
- “全部”表示全部点位，而不是“地图当前可见点位”。
- 筛选只影响地图联动、右侧详情和底部联动区，不扩展为后台管理面板。

## 地图主视图

- 继续使用 `WebView2 + 高德 JSAPI 2.0`。
- marker 继续保持“小图标 + 一行名称”。
- 点击 marker 只联动右侧详情，不弹出大摘要卡片。
- 地图区角落明确显示“当前地图显示 X / 总点位 Y”。
- 同时明确文案：只有具备可用显示坐标的点位会落图。
- 对坐标重合或极近的点位做轻量视觉偏移，不引入复杂聚合框架。

## 右侧详情抽屉

分组顺序固定为：

1. 基础信息
2. 运行态
3. 派单与恢复
4. 坐标与平台
5. 维护信息

“坐标与平台”分组首版至少展示：

- 坐标来源
- 治理状态
- 当前显示坐标
- 平台原始坐标
- 原始坐标类型
- 手工坐标
- 未落图原因
- 治理建议

右侧保留“编辑补充信息 / 补坐标”入口，不新增复杂操作面板。

## 底部联动区

- 左侧继续承载异常缩略条，与地图和详情保持联动。
- 右侧增加轻量“未落图治理”区，不做满屏台账页。
- 未落图治理区展示简要摘要：
  - 点位名称
  - 未落图原因
  - 平台原始坐标摘要
  - 手工坐标摘要
- 点击条目刷新右侧详情，并可直接进入“编辑补充信息 / 补坐标”。

## 编辑弹窗

- 只维护本地补充信息，不改写平台原始数据。
- `deviceCode`、`deviceName` 只读。
- 手工坐标保存后，会重新参与地图落图治理。
- 地图点选只回填手工坐标。
