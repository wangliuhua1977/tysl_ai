# UI 布局职责

## 顶部状态条

- 展示系统名称、点位数、已监测数、异常数、已派单数和最后刷新时间。
- 不提供“新增点位”入口。
- 不展示 SQLite、Stub、日志、调试或诊断信息。

## 左侧筛选栏

- 提供搜索与 `全部 / 异常 / 已监测 / 已派单` 快速筛选。
- 筛选作用于地图点位和底部异常缩略条。
- 不承载平台接入参数或维护表单。

## 中央地图主视图

- 持续承载平台设备快照合并后的地图点位。
- 点位标题遵循“别名优先，设备名称兜底”。
- 地图点击拾取只用于编辑弹窗中的“手工补录坐标”。
- 不允许通过地图拾取表达“新增点位”。

## 右侧详情抽屉

- 展示设备编码、在线状态、监测状态、坐标来源、维护信息、备注和本地更新时间。
- 编辑入口只维护本地补充信息。
- 监测开关属于本地补充信息，不改动平台快照。

## 编辑弹窗

- 只编辑本地补充信息字段。
- `device_code` 与 `device_name` 只读展示。
- 允许首次保存为当前 `device_code` 创建本地补充信息记录。
- 不提供设备新建、平台状态编辑或平台坐标编辑能力。

## 底部异常缩略条

- 展示由平台快照状态驱动的异常、离线、派单或关注点摘要。
- 点击摘要后切换选中点位与右侧详情抽屉。
- 保持轻量，不扩展成任务面板。
