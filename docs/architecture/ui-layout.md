# UI 布局职责

## 顶部状态区

- 展示系统名称、统计卡片和最后刷新时间。
- 展示简化后的平台状态、巡检状态、派单状态。
- 不展示 token、签名、异常堆栈或诊断信息。

## 左侧筛选区

- 提供搜索和 `全部 / 异常 / 已纳管 / 处理中` 快速筛选。
- 只影响地图点位和底部异常缩略条。
- 不增加平台配置按钮或其他重操作入口。

## 地图主视图

- 继续使用 `WebView2 + 高德 JSAPI 2.0`。
- marker 默认只显示别名或设备名，别名优先。
- marker 仅保留极简状态点、轻量派单角标和单行名称标签。
- 摘要卡片默认不展开，仅在 hover / click 时出现。
- 当前选中点位高亮清晰，但不做夸张放大或遮挡地图。

## 右侧详情抽屉

分组顺序固定为：

1. 基础信息
2. 运行态
3. 派单与恢复
4. 坐标与平台
5. 维护信息

说明：

- 保持字段对齐一致。
- 按钮最少化，仅保留“编辑补充信息”“纳入/暂停巡检”和必要时出现的“确认恢复”。
- 不显示过程性说明或调试文案。

## 底部异常缩略条

- 只展示异常、派单、冷却、待恢复、已恢复相关点位。
- 卡片高度、标题、状态标签、副文案统一。
- 与地图选中状态和详情抽屉保持联动。

## 编辑弹窗

- 仅维护本地补充信息。
- `deviceCode`、`deviceName` 只读。
- 地图点选只回填手工坐标，不改动平台原始数据。
