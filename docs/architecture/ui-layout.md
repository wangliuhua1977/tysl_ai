# UI 布局职责

## 顶部状态区

- 展示系统名称、统计卡片和最近刷新时间。
- 展示平台状态、巡检状态、派单状态的收口结果。
- 不显示 token、签名、异常堆栈或诊断信息。

## 左侧筛选区

- 提供搜索和 `全部 / 异常 / 已纳管 / 处理中` 快速筛选。
- 只影响地图点位、右侧详情联动和底部异常缩略条。
- 不新增平台配置按钮或重操作入口。

## 地图主视图

- 继续使用 `WebView2 + 高德 JSAPI 2.0`。
- marker 默认只显示“小图标 + 一行名称”。
- 名称规则为别名优先，设备名兜底，超长文本自动省略。
- marker 仅保留状态色和极小派单角标，不再渲染浮动卡片式名称面板。
- 点击点位只高亮当前点位并刷新右侧详情抽屉，不再弹出摘要面板。
- hover 不出现大面板；地图默认保持点位态势图效果。
- 地图右上角提供克制的风格切换入口。
- 地图风格默认使用“默认地图 / 原生地图样式”，并支持深色科技风、简洁浅色风和灰阶监控风。

## 右侧详情抽屉

分组顺序固定为：

1. 基础信息
2. 运行态
3. 派单与恢复
4. 坐标与平台
5. 维护信息

说明：

- 保持字段对齐一致。
- 仅保留“编辑补充信息”“纳入 / 暂停巡检”和必要时出现的“确认恢复”。
- 不显示过程性说明、调试文案或诊断输出。

## 底部异常缩略条

- 只展示异常、派单、冷却、待恢复、已恢复相关点位。
- 卡片高度、标题、状态标签、副文案统一。
- 与地图选中态和右侧详情保持联动。

## 编辑弹窗

- 只维护本地补充信息。
- `deviceCode`、`deviceName` 只读。
- 地图点选只回填手工坐标，不改动平台原始数据。
