# UI 布局职责

## 顶部状态条
- 负责显示系统名称与全局状态摘要。
- 只承载关键数字和刷新时间，不放复杂操作。

## 左侧筛选栏
- 提供搜索入口和少量快速筛选项，如全部、异常、已监测、已派单。
- 需要支持折叠收起，收起后仍保留核心筛选入口。

## 地图区
- 作为主工作区承载点位分布、状态表达和后续地图接入。
- 当前阶段使用科技感网格背景和假点位卡片占位，后续替换为地图载体。

## 右侧详情抽屉
- 展示当前选中点位的别名、设备名、状态、截图占位和维护信息。
- 复杂操作收入更多菜单或弹出层，不在地图主区扩散。

## 底部异常条
- 承载异常摘要缩略信息，支持后续快速定位、切换和恢复链路。
- 只做一行摘要，不替代完整详情视图。
