# UI 布局职责

## 顶部状态条
- 负责显示系统名称、本地点位摘要、刷新时间和轻量新增入口。
- 不承载大表单或调试信息。

## 左侧筛选栏
- 提供搜索入口和 `全部 / 异常 / 已监测 / 已派单` 快速筛选。
- 支持折叠收起，切换后地图点位与底部异常缩略条同步联动。

## 地图区
- 作为主工作区承载 SQLite 驱动的静态点位和状态表达。
- 当前阶段继续使用科技感占位背景，不接真实地图 SDK。
- 在演示坐标拾取模式下，允许点击当前地图占位区域生成可保存的演示经纬度值。

## 右侧详情抽屉
- 展示当前选中点位的别名 / 设备名、状态、主档字段、维护信息、坐标和备注。
- 编辑入口和监测开关放在抽屉底部，不把表单直接塞回主页面。

## 点位编辑弹窗
- 使用独立 `SiteEditorDialog` 承载新增 / 编辑表单。
- 表单字段覆盖设备编码、设备名称、别名、备注、监测开关、经纬度、地址、产品接入号、维护信息、演示状态和演示派单状态。
- 弹窗内只承载录入和保存，真实地图拾取仍回到主界面地图占位区域完成。

## 底部异常条
- 展示当前筛选范围内的关注点摘要。
- 点击摘要后切换地图选中态与右侧详情抽屉。
