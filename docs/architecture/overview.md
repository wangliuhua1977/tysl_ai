# 架构总览

## 产品定位

`Tysl.Ai` 固定为地图中心型桌面监控系统，工作方式保持为：

- 地图主视图承载点位与状态。
- 右侧详情抽屉承载单点详情、运行态、坐标治理与维护信息。
- 底部联动区承载异常缩略条和轻量未落图治理入口。

系统不回退成传统表格中心页面，也不在主界面暴露调试面板、堆栈或诊断信息。

## 五层边界

- `App`：启动、组合根和依赖注入。
- `UI`：WPF 视图、主题、控件、ViewModel、WebView2 宿主与交互状态。
- `Services`：查询编排、统计汇总、坐标治理判断与 UI 侧用例接口。
- `Infrastructure`：ACIS、SQLite、派单、截图、配置持久化等外部系统接入。
- `Core`：领域模型、枚举和稳定接口。

坐标治理首版仍遵守既有边界：

- 不在 `UI` 里直接做平台签名、坐标转换或 `HttpClient` 调用。
- 不绕过 `AcisApiKernel` 重写 token、签名、解密、坐标转换或预览回退。
- `SiteMapQueryService` 负责聚合“平台原始坐标 / 本地手工坐标 / 当前可落图状态”，不承担真实平台接入细节。

## 当前主链路

1. `App` 启动时读取 `configs/acis-kernel.json`、`configs/amap-js.json`、`configs/dispatch.json`。
2. `AcisKernelPlatformSiteProvider` 通过 `AcisApiKernel` 拉取设备目录与详情。
3. `SilentInspectionHostedService` 驱动 `SilentInspectionService`。
4. 巡检结果落入运行态和截图记录。
5. `DispatchService` 复用运行态完成派单、冷却、恢复与恢复通知。
6. `SiteMapQueryService` 合并平台快照、本地补充信息、运行态和派单记录，并给出：
   - 总点位数
   - 已落图点位数
   - 未落图点位数
   - 未落图原因
   - 未落图治理台账摘要
7. `ShellViewModel` 组织顶部统计、地图主视图、详情抽屉和底部联动区。
8. `AmapHostControl` 与 `UI/Web/amap/*` 负责地图渲染、前端坐标转换和重叠点位轻量偏移。

## V2 第一轮：坐标治理闭环

本轮聚焦“坐标治理与未落图点位管理”，不是新增业务模块。

核心规则：

- 顶部“总点位”统计的是平台快照中的全部点位，不等于地图已显示数量。
- “已落图”表示点位具备可用显示坐标，能够在地图落图。
- “未落图”表示点位当前没有可用显示坐标，必须通过平台数据修正或本地补录继续治理。
- 当前地图区只显示有可用显示坐标的点位。
- 未落图点位通过本地补充信息中的手工坐标继续治理，不新增传统后台台账页。

当前未落图原因首版覆盖：

- 平台未提供坐标，且尚未补录手工坐标。
- 平台坐标不完整，且尚未补录手工坐标。
- 平台原始坐标类型无法识别，且尚未补录手工坐标。
- 手工坐标未补全，暂时无法落图。

## 受控降级

- 缺少 `acis-kernel.json`：应用可运行，但平台点位为空，巡检跳过。
- 缺少 `amap-js.json`：应用可运行，但地图区域显示未配置占位。
- 缺少 `dispatch.json`：应用可运行，但派单链路保持未配置状态。
- 单条 webhook 发送失败：仅影响当前记录，不拖垮其他点位巡检。

## 当前边界

- 不接复杂工单系统、审批流和复杂报表。
- 不扩展视频墙或卡片墙式点位展示。
- 不在 UI 暴露诊断日志、JS 异常或堆栈信息。
- 未落图治理保持轻量入口，不做满屏 DataGrid 后台页面。
