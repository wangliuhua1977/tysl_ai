# 架构总览

## 产品目标
`Tysl.Ai` 面向“天翼视联 AI 巡检系统”重构版桌面端，目标是建立一个地图中心型、可值守、可扩展的监控与巡检工作台，而不是传统后台管理系统。

## 主界面构成
- 顶部状态条：显示系统标题、点位总数、在线数、异常数、已派单数和最后刷新时间。
- 左侧筛选区：放搜索、状态筛选和后续折叠能力。
- 中央地图区：作为主工作区承载点位、状态和后续地图 SDK。
- 右侧详情抽屉：显示单点位详情、截图与操作入口。
- 底部异常缩略条：展示高优先级异常摘要，支持未来快速切换。

## 首版边界
- 本轮只建立解决方案骨架、可运行 WPF 壳、主题资源和占位服务。
- 不接入真实 CTYun、ACIS、地图 SDK、WebView2、SQLite、Webhook。
- 不增加任务面板、调试面板或诊断展示 UI。
- 后续静默巡检、截图优先预览、单点播放和派单闭环均通过分层边界逐轮接入。
