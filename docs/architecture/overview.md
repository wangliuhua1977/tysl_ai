# 架构总览

## 产品目标
`Tysl.Ai` 面向“天翼视联 AI 巡检系统”重构版桌面端，目标是建立一个地图中心型、可值守、可扩展的监控与巡检工作台，而不是传统后台管理系统。

## 主界面构成
- 顶部状态条：显示系统标题、点位总数、纳管数、异常数、已派单数和最后刷新时间。
- 左侧筛选区：放搜索、状态筛选和折叠入口。
- 中央地图区：作为主工作区承载本地点位、状态表达和演示坐标拾取。
- 右侧详情抽屉：显示单点位详情、主档字段、维护信息和操作入口。
- 底部异常缩略条：展示高优先级关注点摘要，支持快速切换。

## 第 2 轮范围
- 本轮完成本地点位主档、SQLite 持久化、地图静态点位、点位编辑弹窗与演示坐标拾取。
- 启动时自动初始化本地数据库，空库时灌入少量演示点位。
- 主页面通过服务层查询本地数据库，再投影为 UI 所需的地图点位与详情 DTO。
- 真实 CTYun、ACIS、地图 SDK、WebView2、Webhook 仍不接入。

## 当前边界
- 不增加任务面板、调试面板或诊断展示 UI。
- 不把平台调用、SQL 访问、坐标演算或数据灌入逻辑写进 UI。
- 后续静默巡检、截图优先预览、单点播放和派单闭环仍通过既定分层逐轮接入。
