---

## 适用场景
- 需要创建或调整地图中心型 WPF 主壳。
- 需要统一主题资源、状态色、卡片样式和布局骨架。
- 需要在不引入真实数据源的前提下搭建高保真占位界面。

## 本技能的执行步骤
1. 先确认主界面是否仍然围绕地图主视图、右侧抽屉和底部异常条展开。
2. 把颜色、间距、字体、按钮和状态色全部收敛到 `ResourceDictionary`。
3. 用最少控件表达顶部状态条、左筛选、地图区、右抽屉和底部异常条。
4. 把说明性文字尽量转移到 `ToolTip`，避免主界面堆说明。
5. 构建并目检是否保持极简、克制、可值守。

## 应避免的问题
- 在页面和控件里散落硬编码颜色值。
- 把地图区做成普通空白面板，没有科技感层次。
- 为了显得丰富而堆叠按钮、角标、诊断信息和长文案。
- 用调试控件污染首屏壳层。

## 产出物清单
- 主壳窗口与 ViewModel。
- 主题资源字典、基础控件样式和状态色定义。
- 地图区、抽屉和异常条的占位布局。

## 验收标准
- UI 可运行且整体风格统一。
- 主题色和状态色可集中替换。
- 主界面保持极简，没有偏离地图中心布局。
