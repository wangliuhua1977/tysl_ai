## 适用场景
- 新仓库刚建立，需要一次性初始化解决方案、项目目录、文档索引和基础构建链路。
- 需要把 README、架构文档、技能目录与项目引用关系统一拉齐。
- 需要在最小可构建前提下建立后续开发边界。

## 本技能的执行步骤
1. 先确认目标产品定位、固定分层、解决方案名称和本轮边界。
2. 使用 `dotnet` CLI 创建解决方案与项目，并补齐明确的项目引用方向。
3. 创建 README、`AGENTS.md`、`.codex/config.toml` 和首批 docs 目录。
4. 清理模板噪声文件，把代码骨架收敛到可运行但不越界的最小状态。
5. 执行 `dotnet build`，只修复当前初始化阶段必须处理的问题。

## 应避免的问题
- 用单项目快速堆页面，绕过既定五层结构。
- 在初始化阶段混入真实平台接入、数据库、地图 SDK 或调试面板。
- 文档与代码边界不一致，导致后续开发时失真。
- 没有构建验证就结束初始化。

## 产出物清单
- 解决方案文件与五层项目。
- 项目级 README、`AGENTS.md`、Codex 配置。
- 架构与平台接入基线文档。
- 首轮可构建的 UI 壳与基础引用关系。

## 验收标准
- `Tysl.Ai.sln` 可成功构建。
- 五层目录、命名空间和引用方向与约束一致。
- 文档、技能和代码边界互相一致，没有真实外部接入残留。
